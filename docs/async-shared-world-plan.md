# 雾铃小镇异步共享风痕方案

## 目标

在不改动当前七天单人主线骨架的前提下，为《雾铃小镇》增加一层“异步共享世界”：

- 玩家在桥边可以留下短句风痕
- 后来的玩家能看到真实玩家留下的布条 / 风记签
- 村民对白只做环境回应，不直接复述玩家原文

## 为什么先做异步，而不是实时联机

当前项目的主线状态、章节推进和自动存档都在前端本地完成，最稳的方式是：

- 保留现有单人叙事和本地存档
- 单独增加一层“共享痕迹数据”
- 让它只影响桥边环境和少量氛围对白

这样改动小、风险低，也更符合雾铃小镇“慢慢被人留住”的气质。

## 这次落地的版本

### 前端

- 新增“桥边共享风痕”信息区
- 新增桥边留痕弹层
- 第五天起开放查看与留痕
- 桥边会渲染最近几条共享风痕的像素纸签 / 布条演出

### 数据层

- 采用 Supabase 作为首选远端存储
- 默认保留本机 fallback，未配置 Supabase 时也能完整试玩与演示 UI
- 配置完成后，同一站点中的玩家会共享桥边痕迹

### 叙事层

- 不重写七天主线
- 只在阿栀、林麦、沈砚的“任务已推进完”的对白里追加环境回应
- NPC 只谈“桥边有陌生人的痕迹 / 多了外乡人的话”，不直接念玩家原文

## 数据结构

建议表：`bridge_traces`

- `id`
- `message`
- `ribbon_color`
- `weather_tag`
- `story_day`
- `visitor_id`
- `author_label`
- `is_visible`
- `source`
- `created_at`

## 内容治理

第一版前端会做这些约束：

- 留言长度限制
- 链接 / 长串数字 / 明显引流格式限制
- 本机发言频率限制
- 不开放自定义昵称

后续如果继续扩展，可以再加：

- 后台审核界面
- RPC / Edge Function 级的频率控制
- 举报与隐藏流程

## Supabase 接入方式

这次实现选择最轻的一层：

- 静态站点直接通过 Supabase Data API 读写
- 使用 `publishable / anon key`
- 安全边界交给 RLS

对应 SQL 模板见 `docs/supabase-bridge-traces.sql`。
